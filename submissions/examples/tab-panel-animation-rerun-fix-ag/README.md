# Tab Panel Animation Replay Fix (Issue #14079)\n\n## What does this do?\nReplaces `animation: ease-kf-fade-in` on tab panels with an `opacity` transition approach that doesn't re-trigger on every `display: none → block` toggle.\n\n## How is it used?\n```html\n<div class="ease-tab-panel">Panel content</div>\n```\n\n## Why is it useful?\nCSS `@keyframes` animations restart every time `display` changes from `none` to `block`. Using `opacity: 0/1` with `transition` instead avoids this restart behavior, preventing the distracting flash when returning to a previously-viewed tab.
